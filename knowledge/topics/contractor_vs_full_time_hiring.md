# Contractor vs Full-Time Hiring

> Buy (contractors) = faster, expensive; Build (full-time) = slower, cheaper; choose based on timeline.

## Framework
- Buying = hiring proven talent already doing the job
- Building = hiring soft skills, training hard skills, 2-3x longer time to productivity

## How-To / Method
- Sporadic work = always contract
- Core function = hire full-time if high-leverage, or contract if you need speed
- Use contractors to test role before hiring full-time

## Example
- Sales training: buy (recruit Amazon affiliates already doing livestream sales) vs build (hire toads, train from scratch)

## Claims
- Contractor hiring is a "buy" decision; full-time hiring is a "build" decision; buying is faster and more expensive, building is cheaper and slower; founders must choose which fits their growth timeline. — [helping_e_commerce_business_owners_scale_8C_6qojTA78.txt]
